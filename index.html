What's happening
under the hood?

Process Control Block

Every process is represented by a PCB — a data structure containing PID, PPID, state, exit status, and an array of child references.

struct PCB { pid, ppid, state, exit_status, children[] }

Process States

A process cycles through four states: RUNNING → WAITING (blocked) → ZOMBIE (exited, uncollected) → TERMINATED (reaped).

RUNNING WAITING ZOMBIE TERMINATED

Fork & Wait

pm_fork() creates a child process. pm_wait() blocks the parent until the child exits, preventing orphaned zombie processes.

child_pid = pm_fork(parent);
pm_wait(parent, child_pid);

Concurrency Model

Worker threads run concurrently, simulated via JS Promises and async/await — mirroring the execution model of a real multithreaded Process Manager.

await pm_wait(pid, child);
